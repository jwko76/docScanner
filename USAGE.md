# PiiScanner 사용 가이드

## 개요

PiiScanner는 PC의 문서·이미지 파일을 고속 스캔하여  
주민등록번호·신용카드번호·전화번호 등 개인정보(PII)를 탐지하고  
Excel·HTML 리포트를 생성하는 도구입니다.

**두 가지 실행 방식**

| 파일 | 설명 |
|------|------|
| `PiiScanner.exe` | CLI 버전 — 명령 프롬프트에서 옵션으로 제어 |
| `PiiScannerUI.exe` | GUI 버전 — 창 UI + 실시간 결과 그리드 |

**주요 특징**
- 순수 C++ 구현 (외부 라이브러리 의존성 없음, 정적 CRT 링크)
- Everything SDK 연동으로 수십만 파일 수초 내 목록화
- Excel + HTML 리포트 자동 생성 (파일 경로 클릭 시 해당 문서 바로 열림)
- Windows OCR API로 이미지 텍스트 추출 (Windows 10 이상)
- Windows IFilter로 Office/PDF/HWP 텍스트 추출
- UTF-8 / EUC-KR 한글 인코딩 자동 감지 (바이트 시퀀스 검증)

## 사전 요구사항

| 항목 | 버전 | 비고 |
|------|------|------|
| Windows | 10 이상 | WinRT OCR 필요 |
| [Everything](https://www.voidtools.com/) | 1.4 이상 | 백그라운드 실행 중이어야 함 (없으면 파일시스템 직접 탐색으로 대체) |
| `Everything64.dll` | SDK | exe와 같은 폴더에 배치 |

### Everything64.dll 준비

1. [voidtools SDK 페이지](https://www.voidtools.com/support/everything/sdk/)에서 `Everything-SDK.zip` 다운로드
2. `Everything64.dll`을 `build/` 폴더에 복사

```
build/
  PiiScanner.exe
  PiiScannerUI.exe
  Everything64.dll   ← 여기에 배치
```

## 실행 — GUI 버전 (`PiiScannerUI.exe`, 권장)


### 화면 구성

```
┌─────────────────────────────────────────────────────────────┐
│ 스캔 경로: [___________________________] [찾아보기]          │
│ 출력 경로: [___________________________] [찾아보기]          │
│ [이미지 건너뜀] 스레드:[0]  최대 크기(MB):[100]             │
│ [▶ 스캔 시작]  [■ 중 지]                                    │
│ [=========== 진행률 ============]                            │
│ 상태: 준비                                                   │
├──────────────────────────────────────────────────────────────┤
│ [로그] [스캔 결과]                                           │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 파일명 │ 탐지 유형 │ 탐지 값 │ 마스킹 │ 줄 │ 맥락      │ │
│ │ ...    │ ...       │ ...     │ ...    │ .. │ ...       │ │
│ └──────────────────────────────────────────────────────────┘ │
│ [HTML 리포트 열기]  [Excel 리포트 열기]                      │
└─────────────────────────────────────────────────────────────┘
```

### 사용 방법

1. **스캔 경로** — 스캔할 폴더 입력 또는 `찾아보기` (비워두면 전체 드라이브)
2. **출력 경로** — 리포트 저장 위치 (기본: exe 폴더)
3. **옵션 설정**
   - `이미지 OCR 건너뜀`: 체크 시 이미지 파일 제외 → 빠른 스캔
   - `스레드`: 병렬 스캔 수 (0 = CPU 코어 수 자동)
   - `최대 크기(MB)`: 이 크기 이상의 파일은 건너뜀
4. `▶ 스캔 시작` 클릭
5. **로그 탭**: 스캔 진행 단계별 메시지 표시
6. **스캔 결과 탭**: 탐지된 개인정보가 실시간으로 그리드에 추가됨
   - 스캔 완료 후 자동으로 이 탭으로 전환
7. **결과 그리드 조작**
   - **행 더블클릭** → 해당 파일 바로 열기
   - **행 우클릭** → 컨텍스트 메뉴
     - `파일 열기`: 기본 연결 프로그램으로 파일 열기
     - `폴더 열기`: 탐색기에서 해당 파일 위치로 이동
8. 완료 후 `HTML 리포트 열기` / `Excel 리포트 열기` 버튼 활성화

## 실행 — CLI 버전 (`PiiScanner.exe`)

```
사용법: PiiScanner.exe [옵션]

  --path <경로>        스캔할 루트 경로 (기본: 전체 드라이브)
  --output <경로>      리포트 저장 폴더 (기본: exe 위치)
  --dll <경로>         Everything DLL 경로 직접 지정
  --skip-images        이미지 OCR 건너뜀 (빠른 스캔)
  --max-size <MB>      파일당 최대 크기 (기본: 100)
  --threads <N>        병렬 스캔 스레드 수 (기본: CPU 코어 수)
  --help, -h           도움말 출력
```

### 예시

```bat
:: 특정 폴더만 스캔
build\PiiScanner.exe --path C:\Users\me\Documents

:: OCR 없이 빠른 스캔, 결과를 D:\Reports에 저장
build\PiiScanner.exe --skip-images --output D:\Reports

:: 파일 크기 50MB 이하만, 스레드 8개
build\PiiScanner.exe --max-size 50 --threads 8

:: Everything DLL 경로 직접 지정
build\PiiScanner.exe --dll C:\Tools\Everything64.dll --path C:\Data
```

## 탐지 유형

| 유형 | 예시 | 검증 방식 |
|------|------|----------|
| 주민등록번호 | `850101-1234566` | 체크섬 + 날짜 유효성 |
| 신용카드번호 | `4532-0151-1283-0366` | Luhn 알고리즘 |

| 이메일 | `hong@example.com` | RFC 패턴 |
| IP 주소 | `192.168.1.100` | 유효 범위 필터 |
| MAC 주소 | `AA:BB:CC:DD:EE:FF` | 패턴 |
| 계좌번호 | `110-123-456789` | 은행 자릿수 패턴 |
| 여권번호 | `M12345678` | 한국 여권 패턴 |
| 운전면허번호 | `11-20-123456-78` | 지역코드 패턴 |
| 한국 주소 | `서울 강남구 테헤란로 123` | 광역시/도 키워드 |

## 출력 파일

스캔 완료 후 출력 폴더에 두 가지 리포트가 생성됩니다.

```
pii_report_20260620_110000.xlsx
pii_report_20260620_110000.html
```

### Excel 리포트 (3개 시트)

| 시트 | 내용 |
|------|------|
| 요약 | 스캔 경로·일시·파일 수·탐지 건수·유형별 통계 |
| 파일 목록 | 파일별 경로(클릭 시 열림)·확장자·탐지 건수·**탐지 유형**·추출 방법·오류 |
| 상세 결과 | 탐지 건별 파일(클릭 시 열림)·유형·마스킹 원문·줄번호·신뢰도·맥락 |

### HTML 리포트 (탭 전환)

| 탭 | 내용 |
|----|------|
| 유형별 통계 | 막대 차트로 유형별 탐지 건수 시각화 |
| 파일 목록 | 파일 경로(클릭 시 열림)·**탐지 유형 배지**·탐지 건수 |
| 상세 결과 | 유형 배지·탐지 값·마스킹·줄번호·맥락 (파일명 클릭 시 열림) |

탐지 텍스트는 자동 마스킹 표시 (예: `850101-*******`)

## 지원 파일 형식

| 범주 | 확장자 |
|------|--------|
| 문서 | `.hwp` `.hwpx` `.doc` `.docx` `.xls` `.xlsx` `.ppt` `.pptx` `.pdf` `.rtf` `.odt` |
| 텍스트 | `.txt` `.log` `.csv` `.tsv` `.xml` `.json` `.html` `.ini` `.cfg` `.conf` |
| 이미지 | `.jpg` `.jpeg` `.png` `.tif` `.tiff` `.bmp` `.gif` `.webp` |

**텍스트 추출 방식**

- 일반 텍스트: 직접 읽기 (UTF-8 / UTF-16 / EUC-KR 자동 감지)
- Office/PDF/HWP: Windows IFilter (설치된 뷰어에 따라 지원 범위 다름)
- OOXML (docx/xlsx/pptx): IFilter 실패 시 ZIP+XML 직접 파싱 fallback
- 이미지: Windows OCR API (ko-KR 우선, 없으면 en-US)

## 보안 참고사항

- `--dll` 경로는 `.dll` 확장자 파일만 허용 (임의 DLL 로딩 방지)
- `--max-size` 값은 1~10,240 MB 범위만 허용
- `--threads` 값은 1~256 범위만 허용
- 스캔 결과 리포트에는 개인정보가 포함되므로 안전한 위치에 저장하세요
- 자세한 보안 설계는 `SECURE_CODING.md` 참조

## 알려진 제한사항

- Everything이 실행 중이지 않으면 파일시스템 직접 탐색으로 대체 (속도 저하 가능)
- HWP/HWPX는 IFilter 설치 여부에 따라 추출 품질 차이 있음
- 이미지 OCR은 인쇄 품질이 낮거나 회전된 이미지에서 정확도 저하
- 암호화된 Office 문서는 추출 불가
- 100MB 이상 파일은 기본적으로 건너뜀 (`--max-size`로 조정 가능)
- `vcpkg/` 폴더가 embedded git repository로 포함됨 — 불필요하면 `git rm --cached vcpkg`로 제거
